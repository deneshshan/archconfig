= Git =

Push sequence

Add => Stage => Commit => Push

Discarding changes

Unstaged changes

Can do git stash This sticks the unstaged changes onto a "stack". The stashed changes can be popped at any time replaying changes at any time. This is useful in replaying certain changes post checkin if you did not want the changes checked in in the first place or on a different branch.

To replace changes: git stash pop

Rollback

Soft

Hard

Change all files locally to match a specific commit: git reset --hard _commit_id_

Inspecting Commits

Show files changed in commit

git show --pretty="format:" --name-only _commit_id_

Show diff of a file in commit

git show _commit_id_
